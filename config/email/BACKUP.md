# Email Backup Strategy

## Overview

Local Maildir (`~/Maildir/`) is backed up daily to pCloud via rclone with
encrypted storage and versioned history. The backup protects against:

- mbsync sync state disasters (mass deletions — happened Feb 2026)
- Exchange retention policy deletions (7-year policy)
- Accidental local file loss

## Architecture

```
~/Maildir/
  ├── work/          ← synced from Exchange via mbsync
  └── local/         ← local-only folders (archives, old backups)

pcloud-crypt:                    ← encrypted mirror of ~/Maildir/
pcloud-crypt:versions/YYYY-MM-DD/ ← files deleted/changed on that date
```

### How it works

- `rclone sync` mirrors `~/Maildir/` to `pcloud-crypt:` daily at 3am
- `--backup-dir` moves any deleted or changed files to a dated version folder
  instead of permanently deleting them
- Versions accumulate forever (only deltas, so growth is modest)
- Encryption uses `pcloud-crypt` remote (crypt overlay on `pcloud:Maildir-backup`)
- Directory names are NOT encrypted (easier to browse/debug)

### Launchagent

`~/Library/LaunchAgents/none.mail.backup.plist` runs `~/.local/bin/backup-mail.sh`
daily at 3am.

### Excludes

- `.mbsyncstate*` — sync state files (regenerated by mbsync)
- `.notmuch/**` — search index (regenerated by `notmuch new`)
- `.DS_Store`

## Disaster Recovery

### Scenario: mbsync mass-deletes emails

1. Stop mbsync immediately
2. Check what was lost:
   ```bash
   rclone ls pcloud-crypt:versions/YYYY-MM-DD/
   ```
3. Restore the deleted files:
   ```bash
   rclone copy pcloud-crypt:versions/YYYY-MM-DD/ ~/Maildir/
   ```
4. Rebuild notmuch index:
   ```bash
   notmuch new
   ```

### Scenario: full local Maildir loss

Restore the entire mirror:
```bash
rclone copy pcloud-crypt: ~/Maildir/
```

### Scenario: need to recover files deleted long ago

Browse version folders:
```bash
rclone lsd pcloud-crypt:versions/
rclone ls pcloud-crypt:versions/2026-01-15/
```

## Archival (retention policy protection)

Exchange has a 7-year retention policy. To protect old messages before
they're deleted server-side (which would cascade via `Expunge Both`):

- `~/.local/bin/archive-old-mail.sh` moves messages older than 6 years
  from `work/` to `local/Work/LSHTM/.YYYY/`
- Messages in `local/` are not synced to Exchange, so retention can't
  touch them
- rclone backs up `local/` to pCloud like everything else

Run monthly (or set up as a launchagent when ready).

## Manual operations

```bash
# Check backup size
rclone size pcloud-crypt:

# Check versions size
rclone size pcloud-crypt:versions/

# List version dates
rclone lsd pcloud-crypt:versions/

# If versions ever get too large, prune old ones
rclone delete --min-age 1y pcloud-crypt:versions/
rclone rmdirs pcloud-crypt:versions/

# Run backup manually
~/.local/bin/backup-mail.sh

# Check backup log
tail -50 ~/.log/backup-mail.log
```
